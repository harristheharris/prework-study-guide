  1. git pull origin main: Before we do anything lets first pull the main branch code to assure we are up to date
    • it could be useful to run a git status to see what branch we are in as well
  2. git chechout -b feature/name: This creates a feature branch that we name
    • Make certain that branch names are relevent
    • git status is not a bad idea either
  3. Make changes to code
  4. git add -A: We stage our changes to be commited
    • Note: this above command stages ALL the changes
  5. git commit -m "desciption of the changes we jut made" This update the feature branch locally aka a commit
    • We should commit as often as possible
